# String Simulation

## 📌 핵심 개념
- 문자열을 실제로 자를 필요는 없음
- **상태(기준 문자, 카운트)** 만 관리하면 되는 문제 다수

---

## 🔑 대표 문제
- Programmers: 문자열 나누기
- Baekjoon: 문자열 폭발, 균형 문자열 문제

---

## 🧠 사고 포인트
- “언제 나눌 것인가”가 핵심
- 자료구조보다 **카운터 기반 시뮬레이션**이 본질인 경우가 많음

---

## ❌ 자주 하는 실수
- 문자열 누적(`+=`)으로 불필요한 비용 발생
- 분리 조건보다 구현에 집중
